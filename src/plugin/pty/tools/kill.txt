Requests termination of a PTY session and optionally removes a confirmed-exited record.

Use this tool to:
- Request that a running process stop
- Clean up a confirmed-exited session
- Remove a session from the list

Usage:
- `id`: The PTY session ID (from pty_spawn or pty_list)
- `cleanup`: If true, removes the record only after termination is confirmed (default: false)

Behavior:
- A stop request is not proof that the process has exited. Check `Termination confirmed` in the response.
- If cleanup=false (default), the session remains in the list with its persisted output intact.
- If cleanup=true, an unconfirmed session remains until a later cleanup call.
- Keeping sessions without cleanup allows you to compare logs between runs

Tips:
- Use cleanup=false if you might want to read the output later
- Use cleanup=true when you're done with the session entirely
- To send Ctrl+C instead of killing, use pty_write with data="\x03"

Examples:
- Request stop but keep logs: cleanup=false (or omit)
- Request stop and remove after confirmation: cleanup=true
